Persistence Options

While Redis lives in memory, it offers robust ways to save data to disk. The choice between RDB, AOF, or a hybrid approach depends on your application's tolerance for data loss versus performance needs. Interact with the diagram below to understand the trade-offs.

RDB Snapshotting

Point-in-time snapshots. Fast, compact, ideal for backups.

AOF Logging

Logs every write operation. More durable, less risk of data loss.

Hybrid (RDB + AOF)

Best of both worlds. Recommended for high durability.

Go Examples

Using go-redis, a high-performance client.

Connection & Basic Ops


ctx := context.Background()
rdb := redis.NewClient(&redis.Options{Addr: "localhost:6379"})
err := rdb.Set(ctx, "user:1:name", "Bob", 0).Err()
val, err := rdb.Get(ctx, "user:1:name").Result()
fmt.Println("name", val) // "name Bob"

Transactions (TxPipeline)


pipe := rdb.TxPipeline()
pipe.Incr(ctx, "counter")
pipe.Expire(ctx, "counter", time.Hour)
_, err := pipe.Exec(ctx)

Publish/Subscribe


// Subscriber
pubsub := rdb.Subscribe(ctx, "news_channel")
msg, err := pubsub.ReceiveMessage(ctx)
fmt.Println(msg.Channel, msg.Payload)

// Publisher
rdb.Publish(ctx, "news_channel", "Hello from Go!")
